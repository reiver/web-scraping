Web Scraping

making the machine-illegible web machine-legible

by Charles Iliya Krempeaux

What is web scraping‽ The intuition might be that — web scraping involves having a machine read a user interface (UI) solely designed and intended for humans that does not include any aids for machines. And, in some ways "pretending" to be a human.

And more definitional answer might be — web scraping is the processes of making the machine-illegible web machine-legible by programmatically extracting specific human-legible (but not machine-legible) data from HTML web pages for alternative use or storage.

The term web scraping evolved from the term screen scraping. Web Scraping used to be known as web screen scraping (a specific type of screen scraping) — “web scraping” is a shortening of “web screen scraping”.

Rule of Thumb

A rule of thumb might be — web scraping involves a machine reading HTML that does not contain: JSON-LD, Microdata, Microformats, RDFa, rel-* semantic HTML, or any other machine-legible formats.

Brittleness

Web Scrapers are notorious for being brittle. What seem like small changes to the user interface (UI) and major breaking changes to a web scraper. A small update to a web page (or web site) can completely break a web scraper.

Brittleness: Byte Offsets

Because web scrapers are trying to read user interfaces (UIs) solely meant for humans (and not machines) — web scrapers have to resort to brittle methods. Such as byte offsets.

Not Web Scraping

If a machine is reading HTML data marked up to make it machine-legible, then it is not web scraping.

If a machine is reading the information in HTML marked up with any of the following, then it is not web scraping: JSON-LD, Microdata, Microformats, RDFa, rel-* semantic HTML, or any other machine-legible formats.

Most XML is a machine-legible. If a machine is reading machine-legible XML, then it is not web scraping.

Most JSON is also a machine-legible format. If a machine is reading machine-legible JSON, then it is not web scraping.

XML, JSON, etc

Formats such as XML, and later JSON, were designed to help remove the need to web scrape. I.e., formats such as XML, JSON, and other machine-legible formats are an alternative to web scrapig.

Screen Scraping

The term web scraping evolved from the term screen scraping.

Screen Scraping comes from the mainframe computer + computer terminal era of computing. Back when applications running on a mainframe computer didn't have an API or machine protocol, and only had a computer terminal based user interface (UI).

Screen Scraping is the process of programmatically reading text from a computer terminal's screen.

ANSI Screen Scraping

When computer terminal emulators running on PCs were starting to replace real (physical) computer terminals, screen scraping accomplished by reading raw ASCII data streams and parsing ANSI escape sequences.

And, like web scraping and other forms of screen scraping, ANSI Screen Scraping was brittle.

DOS Screen Scraping

In the DOS era of the 1980s and 1990s, screen scraping was accomplished by reading the PC's video memory buffer. As, this is where the user interface (UI) was represented.

And, like web scraping and other forms of screen scraping, DOS Screen Scraping was brittle.
