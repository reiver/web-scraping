Web Scraping

making the machine-illegible web machine-legible

by Charles Iliya Krempeaux

Web Scraping is the programmatic extraction of specific human-legible (but not machine-legible) data from web pages that is for alternative use or storage.

The intuition might be that — web scraping involves having a machine read a user interface (UI) solely designed and intended for humans. And, in some ways "pretending" to be a human.

So, for example, reading HTML that does not contain: JSON-LD, Microdata, Microformats, RDFa, rel-* semantic HTML, or any other machine-legible formats. (If a machine is reading HTML data marked up to make it machine-legible, then it is not web scraping.)

Formats such as XML, and later JSON, were design to get rid of the need to web scrape. (Machines reading XML, JSON, or other machine-legible formats is also not web scraping.)

Screen Scraping

The term web scraping evolved from the term screen scraping.

Screen Scraping comes from the mainframe computer + computer terminal era of computing. Back when applications running on a mainframe computer didn't have an API or machine protocol, and only had a computer terminal based user interface (UI).

Screen Scraping is the process of programmatically reading text from a computer terminal's screen.
